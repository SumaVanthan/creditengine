POST /api/v1/normalize

Convert raw CIBIL JSON into normalized internal schema. Parses repayment history strings into monthly DPD block arrays once — the engine never re-parses per rule.

GET /api/v1/companies/{company_id}/rules

List all configured rules for a specific company, including thresholds, operators, scoring weights, and hard reject flags.

POST /api/v1/companies/{company_id}/rules

Add a new rule to a company's configuration. Select a variable template and set the evaluation parameters.

Request Body
{
  "template_id": 1,
  "rule_name": "Bureau Score >= 700",
  "operator": ">=",
  "threshold_value": "700",
  "pass_outcome": "PASS",
  "score_weight": 3.0,
  "score_on_pass": 100,
  "score_on_fail": 0,
  "hard_reject": true
}
template_idintegerReference to one of the 119 variable templates
operatorenum> | >= | < | <=|=| BETWEEN | IN | NOT_IN
threshold_valuestringCutoff value. BETWEEN: "low,high". IN: comma-separated list
pass_outcomeenumPASS or FAIL — what outcome to record when condition is met
score_weightfloatWeight for lead score calculation. Default: 1.0
hard_rejectbooleanIf true, FAIL = immediate decline regardless of total score
PUT /api/v1/companies/{company_id}/rules/{rule_id}

Update an existing rule's threshold, weight, operator, or any other configuration field.

DELETE /api/v1/companies/{company_id}/rules/{rule_id}

Soft-delete a rule by setting active_flag = false. The rule remains in the configuration but is excluded from evaluations.

INFO Lead Score Algorithm

The lead score is computed as a weighted average of all rule outcomes:

Lead Score = SUM(score_weight × score_on_pass/fail) / SUM(score_weight) × 100
Grade A80 – 100Strong profile, recommend approval
Grade B60 – 79Good profile, standard approval
Grade C40 – 59Moderate risk, manual review required
Grade D20 – 39High risk, enhanced due diligence
Grade E0 – 19Decline recommended

Hard Reject Override: Any single hard reject flag results in immediate decline regardless of the total lead score.
